# Security Policy

## Supported Versions

We currently support the following versions with security updates:

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅        |
| < 0.1   | ❌        |

## Reporting a Vulnerability

- Please email security@vamsa.dev with the details of the vulnerability.
- Include steps to reproduce, impact, and any proof-of-concept if available.
- Do **not** open a public issue for security vulnerabilities.

## Response Timeline

- We aim to acknowledge receipt within **72 hours**.
- We will provide an initial assessment within **7 days**.
- We will work with you to confirm the issue, determine impact, and plan remediation.

## Disclosure Policy

- We kindly request that you do not publicly disclose the vulnerability until we release a fix.
- We will coordinate on a disclosure date once a fix is available.

## Scope

- The Vamsa application and associated services within this repository.
- Third-party services are out of scope, but please contact us if you believe they are impacted.

Thank you for helping keep Vamsa and its users safe.
